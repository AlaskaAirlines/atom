Result

public enum Result<Value, Error> where Error : Error

A better way to represent (value: Value?, error: Error?) -> () closure used by many Cocoa APIs.

Because the Result is an enum, both the .failure and .success cases are encouraged to be handled with equal priority by the compiler.

switch result {
    case .failure(let error):
    // Handle `AtomError`.

    case .success(let value):
    // Handle decoded `Model`.
}
