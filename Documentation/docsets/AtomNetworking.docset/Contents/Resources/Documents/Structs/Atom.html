  • cancelAllSessionTasks() Asynchronous

    Cancels all currently running and suspended tasks.

    Calling cancelAllSessionTasks() method will not invalide URLSession instance nor will it empty cookies, cache or credential stores.

    All subsequent tasks will occur on the same TCP connection.

  • enqueue(_:) Asynchronous

    Prepares Service for a network call.

    Calling enqueue(_:) method will not initiate a network call until one of the available methods on Service is called first.

    Note

    Calling enqueue(_:) method multiple times without executing a network call will update previously set requestable property on Service with new value. This framework does not support queue based flow.

    Declaration

    Swift

    public func enqueue(_ requestable: Requestable) async -> Service

  • enqueue(_:expecting:completion:)

    Creates and resumes URLRequest initialized from Requestable.

    Use this method to make network requests where you expect data returned by the service and require that data to be decoded into internal representations - models.

    Note

    This function exists to allow AtomNetworking to offer backward-compatible APIs for applications that want to use the new version of the package without fully transitioning to async/await, thereby continuing to use completion-based APIs.

    Declaration

    Swift

    func enqueue<T>(_ requestable: Requestable, expecting type: T.Type, completion: @Sendable @escaping (Result<T, AtomError>) -> Void) where T : Model

    Parameters

    requestable

    The requestable item containing required data for a network call.
    type

    The type to decode.
    completion

    The completion containing Result where the associated value is either an AtomError or decoded Model instance.
  • enqueue(_:completion:)

    Creates and resumes URLRequest initialized from Requestable.
